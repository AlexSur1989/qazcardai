# Обзор проекта

Проект в папке `C:\Users\user\Desktop\приложение видео` — это SaaS-приложение для генерации изображений, видео и карточек товара.

## Стек

- Next.js 16.2.4, App Router
- React 19
- TypeScript
- Prisma 7 + PostgreSQL
- NextAuth credentials
- Tailwind CSS v4
- BullMQ + Redis для очередей
- Stripe для покупки пакетов токенов
- Kie.ai API для генерации медиа
- S3-совместимое хранилище для файлов

## Основные части проекта

- `src/app` — страницы и API routes.
- `src/components` — UI-компоненты, кабинет, админка, product-card flow.
- `src/lib` — утилиты, env, Prisma, валидации, форматирование, платежные настройки.
- `src/server` — серверные сервисы, очереди, генерация, платежи, модерация.
- `prisma` — схема базы данных и миграции.
- `scripts` — seed-скрипты и worker.
- `qazcard-landing` — отдельный статический лендинг.

## Функциональность

- Регистрация и вход пользователей.
- Личный кабинет с балансом токенов.
- Генерация изображений.
- Генерация видео.
- Сценарий “карточка товара”: исходное фото, классификация категории, концепт-фото, marketplace card, видео.
- История генераций и детальная страница генерации.
- Админка: пользователи, модели, генерации, платежи, webhooks, настройки, промокоды, пакеты токенов.
- Покупка пакетов токенов через Stripe Checkout.
- Webhook Stripe для начисления токенов.
- Webhook Kie для обработки статусов генерации.
- Очередь генераций через Redis/BullMQ.
- Inline-режим очереди для локальной разработки без Redis.

## Проверки

Проверки выполнялись через `npm.cmd`, потому что обычный `npm` в PowerShell заблокирован Execution Policy.

- `npm.cmd run typecheck` — успешно.
- `npm.cmd run lint` — успешно, без ошибок.
- `npm.cmd run build` — успешно вне sandbox.

При первой попытке `npm.cmd run build` внутри sandbox сборка упала с `spawn EPERM`. После запуска вне sandbox production build прошел.

## Важные замечания

### 1. Расхождение в inline-режиме очереди

В проекте есть `QUEUE_MODE=inline`, который нужен для локальной разработки без Redis.

Видео и product-card flow частично поддерживают этот режим, но обычный endpoint генерации изображений `/api/generations/image` все еще требует `REDIS_URL`, `KIE_API_KEY` и `KIE_BASE_URL` вместе.

Файл:

- `src/app/api/generations/image/route.ts`

Это может мешать локальной разработке без Redis именно для генерации изображений.

### 2. Next.js 16: `middleware` устарел

Next.js 16 во время build показывает предупреждение:

`The "middleware" file convention is deprecated. Please use "proxy" instead.`

Нужно мигрировать:

- `src/middleware.ts` -> `src/proxy.ts`
- `export function middleware` -> `export function proxy`

Локальная документация Next.js подтверждает это в:

- `node_modules/next/dist/docs/01-app/02-guides/upgrading/version-16.md`
